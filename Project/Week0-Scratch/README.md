# 作业要求
是时候选择属于你自己的冒险了！你的任务很简单，就是在 `scratch.mit.edu` 时用Scratch实现你选择的任何项目，无论是互动故事、游戏、动画还是其他任何项目，只需满足以下要求：

- 你的项目必须至少使用两个精灵，其中至少一个不能是猫。
- 你的项目必须至少包含三个脚本（即每个精灵不一定有三个）。
- 你的项目必须至少使用一个条件句、至少一个循环和至少一个变量。
- 你的项目必须至少使用你自己制作的一个自定义模块（通过“创建模块”），并且该模块至少需要一个输入。
- 你的项目应该比讲座中演示的大多数内容更复杂（虽然很多讲座内容很有启发性，但都很短），但也可能比Oscartime和常春藤最难的游戏简单一些。


为了满足这些要求，你的项目整体上可能需要几十块拼图碎片。而且你的代码理想情况下不仅要正确，还要设计得很好。
如果你的某个脚本有点长，试着把它拆成多个脚本（每个脚本都有具体功能）。
尽量利用“抽象”：如果你能想象给一组积木（比如喵）起一个描述性的名字，这些块很可能可以移到自定义的积木里！
